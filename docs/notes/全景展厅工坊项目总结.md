## 一、项目背景与产品定位

在数字化展厅、虚拟营业厅等场景需求爆发的背景下，我们需要打造一款 0 代码 SaaS 级 3D 场景搭建平台（全景场景工坊）。

<img src="https://xiaomanbala.oss-cn-beijing.aliyuncs.com/blog/640.gif" controls width="800" />

**核心目标：** 让非技术用户通过「上传全景图 + 配置交互参数」的方式，无需依赖 3D 模型文件，即可自主生成个性化 3D 沉浸式在线展厅。

**业务价值：**
该平台主要应用于：

- 企业展厅和产品展示
- 在线教育和培训
- 文旅景点虚拟展示
- 房地产和室内设计展示

## 一、项目架构
项目分为三个核心模块：

1. 3d-ppt（编辑器）
   
   - 基于Vue 2.x + TypeScript开发的全景内容编辑器
   - 提供可视化的场景编辑、素材管理、交互配置等功能
   - 支持个人和团队协作模式，包含权限管理系统
   - 集成了丰富的素材库和模板系统
2. 3d-ppt-sdk（3D引擎）
   
   - 基于Three.js封装的3D图形引擎库
   - 提供全景渲染、场景管理、动画系统、交互控制等核心功能
   - 支持VR模式、陀螺仪控制、粒子效果等高级特性
   - 采用TypeScript开发，提供完整的类型定义
3. 3d-ppt-template（预览模板）
   
   - 用于展示和播放创建的3D全景内容
   - 提供多种展示模式和交互体验
   - 支持移动端适配和响应式设计

## 二、技术栈选型：为什么是 TS + Vue + Three.js？

项目初面临「多人协作 + 无模型 3D 渲染 + 数据流转复杂」的核心问题，最终选定如下技术栈组合：

```text
TypeScript + Vue2 + Vuex + vue-property-decorator + vuex-modules-decorator + WebSocket + Three.js
```

### 1. 选型核心考量

- **TypeScript：强类型护航**  
  项目体量较大，拆分了用户、权限、作品、3D 渲染等多个核心模块，3–4 名开发者协作时，TS 的强类型检查能提前规避数据类型错误，降低模块间联调成本（尤其是 Three.js 的 3D 坐标、纹理等复杂数据结构）。

- **Vue 生态：成熟且贴合业务**  
  - Vuex：解决多模块跨组件数据共享问题（如 3D 场景参数、用户权限），结合 `vuex-modules-decorator` 实现 TS 友好的模块化状态管理。
  - `vue-property-decorator`：让 Vue2 组件支持 Class 风格写法，适配 TS 的语法习惯。
  - WebSocket：满足 3D 场景编辑过程中实时数据同步的需求（如多人协作调整视角、参数）。

- **Three.js：无模型实现全景图 3D 化的核心**  
  无需依赖 Blender 等工具导出的 3D 模型文件，仅通过全景图即可构建球面 / 柱面 3D 场景，轻量化且降低用户使用门槛，是实现「0 代码 3D 搭建」的核心技术选型。

- **工程化前置：提前封装公共能力**  
  开发初期就完成公共能力的抽离：路由封装、权限基类、统一服务调用层、Three.js 通用渲染方法、公共布局 / 样式，为后续模块开发降低重复工作量。

## 三、核心模块实现：从权限到 3D 渲染的精细化落地

### 3.1 精细化权限控制：页面级 + UI 级双维度

平台涉及「个人 / 企业 2 种身份 + 5 种角色」，不同身份 / 角色需看到不同的页面和功能按钮，因此设计了双层权限体系：

- 页面级权限：动态路由拦截控制访问范围。
- UI 级权限：自定义指令控制元素显隐。

#### （1）页面级权限：动态路由拦截

核心思路：基于用户角色配置可访问路由，通过路由守卫实现权限校验。

```ts
// 1. 角色与路由权限映射
const ROUTER_ROLE_MAP_IDS = [
  { id: 0, value: 'PUBLIC_ROLE' },
  { id: 1, value: 'GROUP_ROLE' },
  { id: 2, value: 'ADMIN_ROLE' },
  { id: 3, value: 'USER_ROLE' },
  { id: 5, value: 'MATERIAL_ROLE' },
  { id: 6, value: 'WORKS_ROLE' }
];

// 2. 根据角色ID匹配权限
const getRole = (id = 0) => {
  const index = ROUTER_ROLE_MAP_IDS.findIndex(item => item.id === id);
  return ROUTER_ROLE_MAP_IDS[index !== -1 ? index : 0].value;
};

// 3. 路由配置中声明权限
const routes = [
  {
    path: 'group',
    name: 'admin-data-group',
    component: () => import('@/modules/manage/admin/data/group.vue'),
    meta: {
      key: 'admin',
      role: 'ADMIN_ROLE', // 仅管理员可访问
      navName: 'admin-data'
    }
  }
];

// 4. 路由守卫拦截权限
router.beforeEach((to, from, next) => {
  const targetRole = to.meta.role || 'PUBLIC_ROLE';
  if (UserModule.userId) {
    // 已登录：校验角色权限
    UserModule.roles.includes(targetRole) ? next() : next('/home');
  } else {
    // 未登录：仅放行无权限要求的页面
    !to.meta.role ? next() : next('/home');
  }
});
```

#### （2）UI 级权限：自定义指令控制元素显隐

针对按钮、链接等 UI 元素，开发权限指令，通过「用户角色权限点」与「UI 元素所需权限」的比对，控制元素是否渲染：

```ts
// 1. 提取当前用户的权限操作点
const getActions = () => UserModule.role_actions.map(action => action.value);

// 2. 权限校验方法
export const checkPermissionAction = (
  value: string|string[], 
  type: 'some'|'every' = 'some'
): boolean => {
  if (!value || value.length === 0) return true;
  const permissions = Array.isArray(value) ? value : [value];
  const actions = getActions();
  return permissions[type](ac => actions.includes(ac));
};

// 3. 注册权限指令
Vue.directive('permission', {
  inserted(el, binding) {
    const hasPermission = checkPermissionAction(binding.value);
    if (!hasPermission) el.parentNode?.removeChild(el);
  }
});

// 4. 页面中使用
// <button v-permission="['works:edit']">编辑作品</button>
```

### 3.2 作品管理模块：动态缓存与数据隔离

作品管理是核心业务模块，需支持「个人 / 团队身份切换」「作品权限隔离」「页面缓存动态控制」三大核心能力：

- **数据隔离：身份维度的作品展示**  
  - 个人身份：展示私有作品（上传的全景图、配置的 3D 参数）。
  - 团队身份：展示团队内作品（普通成员仅可操作自己创建的内容，可使用他人发布的团队作品）。

- **动态 Keep-Alive：按需缓存页面**  
  场景：从作品管理进入素材关联页（全景图上传 / 选择），返回时需缓存作品管理页；从作品管理进入其他页面则无需缓存。

```vue
<template>
  <transition name="fade-transform" mode="out-in">
    <keep-alive :include="cached">
      <router-view :key="$route.fullPath" />
    </keep-alive>
  </transition>
</template>

<script lang="ts">
import { Vue, Watch } from 'vue-property-decorator';
import { Route } from 'vue-router';

export default class Layout extends Vue {
  cachedArr: string[] = [];

  get cached() {
    return this.cachedArr.join(',');
  }

  @Watch('$route')
  onRouteChange(to: Route, from: Route) {
    if (!from.meta.componentName) return;
    // 进入素材关联页（全景图上传）：缓存来源页
    if (to.name === 'relation-material') {
      !this.cachedArr.includes(from.meta.componentName) && this.cachedArr.push(from.meta.componentName);
    } else {
      // 进入其他页：移除缓存
      const idx = this.cachedArr.indexOf(from.meta.componentName);
      idx > -1 && this.cachedArr.splice(idx, 1);
    }
  }
}
</script>
```
### 3.3 核心：Three.js 实现全景图 3D 渲染（无模型依赖）
这是平台的核心技术模块，无需 3D 模型文件，仅通过用户上传的全景图，即可生成可交互的 3D 沉浸式场景，核心分为「全景图加载→3D 场景构建→交互控制」三步。
- （1）核心原理 
全景图本质是一张等距圆柱投影 / 球面投影的图片，Three.js 通过创建球面几何体（SphereGeometry） ，将全景图作为纹理贴到球面内侧，再将相机置于球面中心，即可实现「沉浸式 3D 视角」，用户可通过鼠标 / 触控操作旋转、缩放视角。
- （2）实现步骤（核心代码）

```typescript
import * as THREE from 'three';
import { OrbitControls } from 'three/examples/jsm/controls/OrbitControls';

export default class Panorama3DRenderer {
  // 核心对象
  private scene: THREE.Scene;
  private camera: THREE.PerspectiveCamera;
  private renderer: THREE.WebGLRenderer;
  private controls: OrbitControls;
  private sphereMesh: THREE.Mesh; // 全景图球面网格

  // 初始化3D场景
  init(container: HTMLElement, panoramaUrl: string) {
    // 1. 创建场景
    this.scene = new THREE.Scene();

    // 2. 创建相机（置于球面中心）
    this.camera = new THREE.PerspectiveCamera(
      75, // 视角
      container.clientWidth / container.clientHeight, // 宽高比
      0.1, // 近裁剪面
      1000 // 远裁剪面
    );
    this.camera.position.set(0, 0, 0); // 相机在球面中心

    // 3. 创建渲染器
    this.renderer = new THREE.WebGLRenderer({ 
      antialias: true, // 抗锯齿
      alpha: true 
    });
    this.renderer.setSize(container.clientWidth, container.clientHeight);
    this.renderer.setPixelRatio(window.devicePixelRatio); // 适配高清屏
    container.appendChild(this.renderer.domElement);

    // 4. 加载全景图纹理，创建球面网格
    const textureLoader = new THREE.TextureLoader();
    textureLoader.load(
      panoramaUrl,
      (texture) => {
        // 球面几何体（半径500，分段数提高渲染精度）
        const sphereGeometry = new THREE.SphereGeometry(500, 60, 40);
        // 材质：全景图纹理，贴到球面内侧（side: THREE.BackSide）
        const sphereMaterial = new THREE.MeshBasicMaterial({
          map: texture,
          side: THREE.BackSide // 内侧可见（相机在球面内）
        });
        // 创建网格并加入场景
        this.sphereMesh = new THREE.Mesh(sphereGeometry, sphereMaterial);
        this.scene.add(this.sphereMesh);
      },
      (progress) => {
        // 加载进度（用于展示loading）
        console.log(`加载进度：${(progress.loaded / progress.total) * 100}%`);
      },
      (error) => {
        console.error('全景图加载失败：', error);
      }
    );

    // 5. 添加轨道控制器（实现视角交互）
    this.controls = new OrbitControls(this.camera, this.renderer.domElement);
    this.controls.enableDamping = true; // 阻尼效果，更顺滑
    this.controls.dampingFactor = 0.05;
    this.controls.maxPolarAngle = Math.PI / 2; // 限制垂直视角，避免穿模

    // 6. 监听窗口大小变化，自适应渲染
    window.addEventListener('resize', this.onWindowResize.bind(this));

    // 7. 启动渲染循环
    this.animate();
  }

  // 渲染循环
  private animate() {
    requestAnimationFrame(this.animate.bind(this));
    this.controls.update(); // 更新控制器
    this.renderer.render(this.scene, this.camera);
  }

  // 窗口大小自适应
  private onWindowResize() {
    const container = this.renderer.domElement.parentElement;
    if (!container) return;
    this.camera.aspect = container.clientWidth / container.clientHeight;
    this.camera.updateProjectionMatrix();
    this.renderer.setSize(container.clientWidth, container.clientHeight);
  }

  // 切换全景图
  changePanorama(panoramaUrl: string) {
    const textureLoader = new THREE.TextureLoader();
    textureLoader.load(panoramaUrl, (texture) => {
      this.sphereMesh.material.map = texture;
      this.sphereMesh.material.needsUpdate = true; // 强制更新材质
    });
  }

  // 销毁3D场景（避免内存泄漏）
  destroy() {
    window.removeEventListener('resize', this.onWindowResize.bind(this));
    this.controls.dispose();
    this.renderer.dispose();
    this.scene.remove(this.sphereMesh);
    this.sphereMesh.geometry.dispose();
    (this.sphereMesh.material as THREE.MeshBasicMaterial).map?.dispose();
    this.sphereMesh.material.dispose();
  }
}
```

## 四、性能优化实战：针对全景图 + Three.js 的专项优化

### 4.1 先分析：找准性能瓶颈

通过三大工具定位问题，重点关注 Three.js 渲染相关瓶颈：

| 工具        | 核心关注点                      | 发现的问题                                               |
| ----------- | ------------------------------- | -------------------------------------------------------- |
| Network     | 全景图加载耗时、JS 加载顺序     | 全景图体积大导致加载慢、Three.js 核心库阻塞首屏         |
| Performance | 渲染帧率、JS 执行耗时           | 全景图纹理加载导致主线程阻塞、渲染循环出现掉帧           |
| Lighthouse  | 3D 场景交互流畅度、资源加载     | 首屏 LCP 高、WebGL 渲染占 GPU 过高                      |

### 4.2 针对性优化方案

#### （1）全景图资源优化

- 压缩与格式转换：限制用户上传全景图分辨率（建议 4096×2048），后端自动将图片转为 WebP 格式，体积减少 50% 以上。
- 懒加载 + 预加载：非首屏全景图延迟加载，首屏全景图通过 `link rel="preload"` 预加载，且优先加载低分辨率缩略图占位，再渐进式加载高清图。
- 纹理复用：相同全景图复用 Three.js 纹理对象，避免重复加载。

#### （2）Three.js 渲染优化

- 几何体分段数适配：根据设备性能动态调整球面几何体的分段数（高端设备 60×40，低端设备 30×20），平衡画质与性能。
- 关闭不必要的渲染特性：仅使用 `MeshBasicMaterial`（无需光照），关闭阴影、部分抗锯齿（低端设备）等非必要特性。
- 渲染循环节流：非交互状态下降低渲染帧率（从 60fps 降至 30fps），减少 GPU 占用。
- 内存管理：组件销毁时彻底释放 Three.js 的几何体、纹理、渲染器等资源（参考上文 `destroy` 方法），避免内存泄漏。

#### （3）JS 加载优化

- Three.js 按需加载：通过动态 `import` 加载 Three.js 核心库和控制器，非 3D 编辑页面不加载。
- 拆分长任务：将全景图纹理加载、几何体创建等逻辑拆分为微任务，避免阻塞主线程。
- 限制高频交互：对鼠标 / 触控的视角操作添加节流，避免短时间内多次触发渲染更新。

## 五、开发踩坑与解决方案


### 5.1 音频播放：缓存就绪后再执行播放逻辑

3D 场景中常需搭配背景音频，直接调用 `play()` 易因音频未缓存完成导致失败：

```js
// 修复前：直接播放，易失败
if (this.audio.src !== url) {
  this.audio.src = url;
}
this.audio.paused ? this.audio.play() : this.audio.pause();

// 修复后：监听canplay事件，缓存就绪后播放
if (this.audio.src !== url) {
  this.audio.src = url;
  this.audio.addEventListener("canplay", function () {
    this.play();
  }, false);
} else {
  // 同一音频地址：直接切换播放/暂停
  this.audio.paused ? this.audio.play() : this.audio.pause();
}
```

### 5.2 Three.js 常见踩坑

- 视角穿模：限制 `OrbitControls` 的 `maxPolarAngle` 和 `minDistance`，避免相机超出球面范围。
- 纹理拉伸：确保全景图比例为 2:1（等距圆柱投影），加载时设置 `texture.wrapS = THREE.RepeatWrapping`。
- 移动端适配：根据设备性能调整 `controls` 配置，避免触控操作卡顿，并适配视口大小。

## 六、工程化与可维护性提升

### 6.1 接口服务统一管理

将接口按模块封装为 Class，结合 TS 的类型提示，便于维护和修改（尤其是全景图上传、3D 参数保存等接口）：

- 按业务模块拆分服务类（如 `UserService`、`WorksService`、`PanoramaService`）。
- 统一请求拦截、响应处理、错误捕获。
- TS 类型约束：全景图上传参数、3D 场景配置参数等类型化，修改接口时自动提示调用处错误。

### 6.2 代码规范与复用

- 拆解 Three.js 渲染逻辑为独立类，与 Vue 组件解耦，便于复用和测试。
- 提取 3D 交互通用方法（视角重置、全景图切换、分辨率适配）为工具函数。
- 接入 ESLint / StyleLint：统一代码风格，降低多人协作成本。
- 完善注释：对 Three.js 核心方法、权限校验逻辑添加详细注释，便于后期维护。
