# 可视化编辑器

## 项目概述

本项目是一个基于Vue.js和Three.js开发的可视化编辑器，旨在为数据中心提供直观的3D可视化管理和监控解决方案。该编辑器支持2D/3D双模式编辑，允许用户通过拖拽组件、配置属性、设置数据绑定等方式，快速构建可视化监控界面。

<video src="https://xiaomanbala.oss-cn-beijing.aliyuncs.com/blog/visual-editor-platform.mp4" controls width="800" />

## 技术架构

### 前端技术栈
- **框架**: Vue 2.7 + TypeScript
- **3D渲染引擎**: Three.js
- **图形编辑**: AntV X6（用于2D图形编辑和流程图）
- **UI组件库**: Element UI
- **状态管理**: Vuex + Vuex Module Decorators
- **构建工具**: Vue CLI 5.0
- **代码规范**: ESLint + Prettier

### 核心架构设计

项目采用模块化架构，主要分为以下几个核心模块：

1. **屏幕编辑器模块**：负责2D屏幕的编辑和组件管理
2. **3D场景模块**：基于Three.js实现的3D场景渲染和交互
3. **数据管理模块**：处理组件数据绑定和API请求
4. **组件库模块**：提供可复用的可视化组件
5. **蓝图编辑模块**：实现可视化逻辑编程

## 核心功能实现

### 1. 双模式编辑器

项目实现了2D和3D两种编辑模式的无缝切换：

```typescript
// 项目状态管理
export enum ProjectState {
    BUILD = 0,  // 编辑模式
    PREVIEW = 1, // 预览模式
    BLUE = 2,    // 蓝图模式
}
```

2D模式主要用于布局设计和组件配置，而3D模式则提供机房空间的真实感展示。用户可以在两种模式间自由切换，实现从平面设计到立体空间的完整构建。

### 2. 组件化架构

项目采用高度组件化的设计，所有可视化元素都继承自基础对象类：

```typescript
export class ObjectBase extends ObjectEvent {
    objectType = ObjectType.BASE
    defaultVisible = true
    _visible = true
    _active = false
    _parent: ObjectGroup | Object2dScene | null = null
    _children: (ObjectElement | ObjectGroup)[] = []
    
    // 位置和尺寸管理
    position = new Vector2(0)
    size = new Vector2(1, 1)
    // 锁定组件
    lock = false
    // 重命名编辑模式
    editing = false
}
```

通过这种继承体系，实现了组件的统一管理和操作，如选择、移动、缩放、复制粘贴等。

### 3. 3D场景渲染系统

3D渲染系统基于Three.js构建，采用了分层设计：


这种设计使得3D场景的各个部分可以独立管理和配置，提高了代码的可维护性和扩展性。

### 4. 数据绑定与交互

项目实现了灵活的数据绑定系统，支持静态数据、API接口数据等多种数据源：

## 技术难点与解决方案

**难点**：如何在2D编辑器和3D场景之间保持数据同步和状态一致？

**解决方案**：
- 设计统一的数据模型，通过JSON序列化实现数据持久化
- 实现事件驱动的更新机制，确保2D和3D视图的实时同步
- 使用观察者模式管理组件状态变化

```typescript
// 场景切换逻辑

```

- 实现视锥体剔除和遮挡剔除优化

## 应用价值

该编辑器已成功应用于多个IDC机房的可视化监控项目，实现了：

- **机房资源可视化**：直观展示机柜布局、设备状态和空间利用情况
- **实时监控集成**：对接各类监控系统，实现数据可视化展示
- **运维效率提升**：通过可视化界面简化运维操作流程
- **规划决策支持**：为机房扩容和设备布局提供直观的规划工具
